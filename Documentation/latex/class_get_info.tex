\doxysubsection{Detailed Description}
This script is used to grab the information that\textquotesingle{}s neccessary from the \char`\"{}\+Data\+Container\char`\"{} script. 

The script is attached to the \char`\"{}\+Data\+Point\char`\"{} gameobject. It will then deteced if the user has clicked onto the node. If so, then it\textquotesingle{}ll populate the \char`\"{}\+Body\char`\"{} of the \char`\"{}\+Nodemenu\+Panel\char`\"{} of the Nodemenu prefab. This script is in-\/progress to fix it\textquotesingle{}s behaviour. The core problem is that when the nodemenu is active and if there\textquotesingle{}s a node behind the menu, the click gets detected through the button on the menu, if there\textquotesingle{}s another node in-\/line with that button. For example, if you\textquotesingle{}ve already clicked on a node and the nodemenu has opened, when you try to click on \char`\"{}\+View Details\char`\"{} button right where you know there\textquotesingle{}s is a node in-\/line/directcly behind the button, the nodemenu of that node will get activated. Also the user should be able to click-\/hold and drag the node if they want to and be able to just click on the node to open the nodemenu. This functionality is still I\+N-\/\+P\+R\+O\+G\+R\+E\+SS (not a complete solution). One fix that was being attempted was to count the seconds the user holds the node for, and if it\textquotesingle{}s greater then two, then make it ignore the activation of the nodemenu and vice-\/versa for the other scenerio.
